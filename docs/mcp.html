MCP Server

Every API in your catalog is automatically available as a tool for AI agents. Connect Claude Desktop, Claude Code, or any MCP-compatible client and let AI call paid APIs directly — with automatic USDC payment per call.

What is MCP?

Model Context Protocol (MCP) is an open standard that lets AI assistants call external tools and data sources. When your control center is connected as an MCP server, Claude (and other AI agents) can search your catalog, call APIs, and use the results directly in conversation — without any copy-pasting or manual steps.

How it works

Your control center runs an MCP server on port 3003. It starts automatically when your catalog loads. Each API in your catalog becomes a separate MCP tool — one tool per endpoint (GET, POST, etc.).

When an AI agent calls a tool:

  1. The agent sends a tool call request to your MCP server (HTTP POST to /mcp)
  2. Your control center routes the call to the API provider
  3. USDC is deducted from your wallet automatically (same as any API call)
  4. The response is returned to the agent in structured JSON

No code changes needed. The MCP server reads your existing catalog. Every API you subscribe to is immediately available as a tool. New APIs appear automatically after each catalog sync.

Authentication

The MCP server uses Bearer token auth — the same consumer secret that protects all your service calls.

To find your consumer secret:

  1. Open your control center panel
  2. Go to Settings → Developer
  3. Copy the Consumer Secret

The secret looks like: ops_cs_64-character-hex-string

You can rotate it at any time. After rotating, regenerate all SDKs and update your MCP client config.

Connecting Claude Desktop

Claude Desktop supports MCP servers over HTTP. Add your control center to Claude Desktop's config file:

Config file location:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json
{
  "mcpServers": {
    "sertone": {
      "type": "http",
      "url": "http://localhost:3003/mcp",
      "headers": {
        "Authorization": "Bearer ops_cs_YOUR_CONSUMER_SECRET"
      }
    }
  }
}

Replace ops_cs_YOUR_CONSUMER_SECRET with your actual consumer secret. If your control center is on a remote machine, replace localhost with its IP address.

After saving the config, restart Claude Desktop. A hammer icon in the chat window confirms tools are loaded.

Connecting Claude Code

Claude Code (the CLI) can also connect to your MCP server. Add it to your project's .claude/settings.json:

{
  "mcpServers": {
    "sertone": {
      "type": "http",
      "url": "http://localhost:3003/mcp",
      "headers": {
        "Authorization": "Bearer ops_cs_YOUR_CONSUMER_SECRET"
      }
    }
  }
}

Or add it globally with:

claude mcp add --transport http sertone \
  http://localhost:3003/mcp \
  --header "Authorization: Bearer ops_cs_YOUR_CONSUMER_SECRET"

What tools appear

Each API endpoint in your catalog becomes a separate MCP tool. The tool name is derived from the API name and operation ID:

APIEndpointTool name
Weather APIGET /forecastWeather_API_getForecast
Weather APIPOST /alertsWeather_API_postAlerts
Crypto PricesGET /price/{symbol}Crypto_Prices_getPrice

Each tool's description shows the API name, endpoint description, HTTP method, path, and price per call:

[Weather API] Get 5-day forecast by city | GET /forecast | $0.001 USDC/call

Tool parameters

Tool parameters are extracted from the API's OpenAPI specification automatically:

  • Path parameters (e.g., {symbol} in /price/{symbol}) — required
  • Query parameters — required or optional as defined in the spec
  • Request body (POST/PUT) — included as a body parameter with the full JSON Schema

The AI agent reads the parameter schema and fills in values from the conversation context, exactly as a developer would.

Example conversation

With the MCP server connected, a conversation like this works automatically:

1
You ask: "What's the current price of Bitcoin and the 5-day weather forecast for Tokyo?"
↓
2
Claude calls two tools: Crypto_Prices_getPrice({ symbol: "BTC" }) and Weather_API_getForecast({ city: "Tokyo", days: 5 })
↓
3
Your control center: routes both calls, deducts USDC from your wallet, returns results
↓
4
Claude responds: with a complete answer using live data — total cost: $0.002 USDC

Subscription MCP (for API providers)

If you sell APIs, you can offer MCP access to your customers as a monthly subscription. This runs on port 3005 and adds a subscription management layer on top of the standard call-per-use model.

Customers subscribe once with a token payment, then make unlimited MCP calls within their subscription window:

# Subscribe (one-time, monthly)
POST http://your-control-center:3005/subscribe
{ "api_id": "YOUR_API_ID", "duration_days": 30, "token": "USDC_TOKEN" }

# Check subscription status
GET http://your-control-center:3005/subscription/status?api_id=YOUR_API_ID

# Call any API endpoint
GET http://your-control-center:3005/api/YOUR_API_ID/your/endpoint?param=value

Auto-discovery

Your control center publishes a discovery manifest at:

http://your-control-center/.well-known/claude.json

AI agents and marketplaces that support auto-discovery can find your MCP endpoint and its capabilities without manual configuration.

Frequently asked questions

Does the MCP server start automatically?

Yes. It starts when your catalog loads for the first time after boot. If your catalog is empty, the server starts as soon as the first API is registered.

Can I disable the MCP server?

Yes. Set MCP_ENABLED=false in your docker-compose environment. The server will not start.

Can I change the port?

Yes. Set MCP_PORT=XXXX in your docker-compose environment. Default is 3003.

Is the MCP server available on the public internet?

That depends on your network configuration. By default, the control center binds to all interfaces. If you want MCP-only access for local AI agents, restrict port 3003 in your firewall rules.

What happens if an API call fails?

The tool returns an error result with the failure reason. No USDC is charged for failed calls. The AI agent sees the error and can choose a different approach.

Does streaming work with MCP?

Yes. GET /mcp on the MCP server returns an SSE stream for real-time notifications. Streaming API results (for long-running AI inference APIs) work through the subscription port (3005) via POST /stream/:apiId.
