Sertone Documentation

Everything you need to offer and consume services on the Sertone network.

What is Sertone?

Sertone is a global service network. There are no Sertone servers. There is no Sertone database. There is no Sertone employee approving your listing or processing your payment.

Every participant runs their own Sertone control center — a single Docker container on their own hardware. Sellers register services. Buyers discover them. Calls flow directly between participants through the Sertone Global Network, a proprietary fully encrypted protocol. Payment settles automatically in USDC.

Nobody controls who can join. Nobody controls what you sell. Nobody can see what you buy. Nobody can freeze your funds.

Sertone built the protocol — the roads, not the shops.

The un-marketplace

Traditional API marketplaces charge 20–40%, control your listings, see your traffic, and can ban you at will. Sertone is the un-marketplace.

What services can you offer?

  • REST APIs — standard HTTP endpoints
  • GraphQL — query-based, nested data
  • SOAP & XML-RPC — legacy enterprise protocols
  • gRPC — high-performance binary protocol
  • JSON-RPC — lightweight remote procedure calls
  • Live Streams — SSE, WebSocket, video feeds
  • Smart Contracts — any function on any EVM blockchain, served as REST
  • Webhooks — asynchronous event delivery

Every protocol. Every format. One network. One settlement.

How it works

  1. Install — docker run sertone/wrapper:latest on any machine
  2. Register — add your service via the web console, set your price
  3. Publish — your service appears in the global catalog automatically
  4. Earn — consumers call your service, USDC settles automatically. You earn 95%.

You keep 95%. Sertone deducts a 5% protocol royalty from every settlement — automatically, with no invoices, no approval process, and no payment terms. There is no other fee.
