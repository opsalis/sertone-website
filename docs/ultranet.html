Mesh Browsing

The Mesh tab in your catalog lets you reach private services on the Sertone Global Network — without your IP address ever being seen by the service provider, the gateway operator, or anyone in between.

What is Mesh Browsing?

When you use the Sertone catalog, your requests normally go to services hosted on the open internet — REST endpoints, GraphQL APIs, webhook targets. Mesh Browsing is different: it lets you reach services that live inside the Sertone Global Network and are never exposed to the public internet at all.

These private services have addresses in the .gtw namespace. They can only be reached through the Sertone network. Nobody outside the network can discover or connect to them directly.

From a consumer's perspective, it feels like a browser. You type an address. You get a response. But under the surface, your request travels through multiple encrypted hops across the global network before reaching the service — and the service never learns who you are or where you came from.

How to Use the Mesh Tab

  1. Open your catalog page — the one served from your gateway on port 80.
  2. Click the Mesh tab at the top of the page.
  3. In the address bar, type a .gtw address. Examples:
    • payments.gtw — access on default port 80
    • analytics.gtw:8080 — access on a specific port
    • media.gtw:443 — HTTPS service behind the mesh
  4. Click Fetch. The response appears below — HTML, JSON, or raw text depending on the service.
Privacy guarantee: The service you reach sees nothing about you. Not your IP address, not your gateway's IP address, not your country. The only information it receives is the request itself.

Understanding .gtw Addresses

Addresses in the .gtw namespace are registered by service providers on the Sertone network. A provider maps a name like myservice.gtw to one of their gateways, and then adds port routes that define which backend their gateway proxies requests to.

Unlike regular domain names, .gtw names are:

  • Private by design — they cannot be looked up with regular DNS. They only resolve inside the Sertone network.
  • Censorship-resistant — no registrar can revoke them. No government can block them. They exist only in the network's shared registry.
  • Self-hosted — the provider runs the backend themselves on their own hardware. Sertone is just the routing layer.

Hosting a .gtw Service (Providers)

To offer a service under a .gtw address, you need two things:

1. Register a domain

In your gateway's admin panel, go to Mesh → Domains and register a name. You choose the label (e.g. payments) and the TLD is always .gtw. Registration is tied to your gateway's identity on the network — only your gateway can serve that name.

2. Add port routes

A port route tells your gateway what to do when a consumer requests payments.gtw:80. You map the port to a backend address reachable from your gateway's server. For example:

Mesh AddressBackend
payments.gtw:80localhost:3000 — your local web app
payments.gtw:443192.168.1.10:443 — a server on your LAN
payments.gtw:808034.2.5.2:8080 — any TCP address your server can reach

The backend is not exposed to the internet. It only needs to be reachable from the machine where your gateway runs.

Pricing

When adding a port route, you can set a price per request in USDC and choose a billing model (per-use or subscription). The Sertone settlement layer handles payment automatically before routing each request to your backend.

Security Model

The Sertone Global Network uses end-to-end encryption between the consumer's gateway and your backend. Intermediate nodes in the routing path can see that a request is passing through, but cannot read the payload or determine its origin or destination.

Additional protections built into the routing layer:

  • Backend addresses cannot be private IPs or localhost — the routing layer enforces this for all providers, preventing internal network attacks.
  • Sensitive ports (SSH, database ports, admin ports) are blocked from being registered as backends.
  • All consumer requests are rate-limited per gateway to prevent abuse.
  • Only valid .gtw addresses can be requested — arbitrary internet addresses cannot be reached through the Mesh tab.

Frequently Asked Questions

Can I browse regular websites through the Mesh tab?

No. The Mesh tab is specifically for .gtw services hosted on the Sertone network. It is not a general-purpose internet browser. Regular HTTP/HTTPS sites are not accessible through this tab.

How is this different from a VPN?

A VPN hides your IP from the destination but the VPN operator can see everything. With Sertone Mesh Browsing, the routing is multi-hop and encrypted so no single participant — including your own gateway — can see both who you are and what you are requesting.

What response types are supported?

Any TCP payload — HTML, JSON, XML, binary data. The Mesh tab displays text responses directly. For binary content, the raw data is available. The service on the other end can serve any content type it supports.

Is there a size limit on responses?

Yes — responses are capped at 10 MB per request to protect network participants from resource exhaustion. This is a hard limit enforced by all gateways.

What happens if the service is offline?

If the provider's gateway is unreachable, you receive an error message. The domain registration remains valid and the service resumes automatically when the provider's gateway comes back online.
