Mesh Browsing

Private Services. No IP Exposure. Anywhere.

Access services hosted inside the Sertone Global Network — through an anonymous multi-hop path that hides your identity from every participant, including the service provider.

Live Demo — This demo runs on a production cluster of Sertone gateways. When you fetch a .gtw service below, your request travels through the live network to a real service running on a gateway you do not know the location of.

Services under the .gtw domain have no public IP, no public DNS record, and no exposure to the internet. They are only reachable through the Sertone network.

Calling .gtw Services from Your Code

Once your Sertone is running, you can access .gtw services through its local API:

# Fetch a .gtw service through your local Sertone
$ curl -X POST http://localhost/api/mesh/fetch \
  -H "Content-Type: application/json" \
  -H "X-Mesh-Token: YOUR_SESSION_TOKEN" \
  -d '{"url": "demo.gtw"}'

# Response example
{
  "ok": true,
  "status": 200,
  "contentType": "text/html",
  "body": "<html><body>...</body></html>"
}

# Specify a port
$ curl -X POST http://localhost/api/mesh/fetch \
  -d '{"url": "payments.gtw:8080"}'
// Fetch a .gtw service from the Mesh tab in your Sertone catalog
// Your session token is embedded in the page when your Sertone serves it
async function meshFetch(address) {
  const r = await fetch('http://localhost/api/mesh/fetch', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Mesh-Token': window.MESH_TOKEN  // injected by your Sertone
    },
    body: JSON.stringify({ url: address })
  });
  const data = await r.json();
  if (!data.ok) throw new Error(data.error);
  return data.body;
}

// Usage
const html = await meshFetch('demo.gtw');
document.getElementById('result').innerHTML = html;

// Fetch JSON API
const json = await meshFetch('api.gtw:8080');
console.log(JSON.parse(json));
# Python — access .gtw services via your local Sertone
import requests

def mesh_fetch(address, token):
    r = requests.post(
        'http://localhost/api/mesh/fetch',
        json={'url': address},
        headers={'X-Mesh-Token': token}
    )
    data = r.json()
    if not data['ok']:
        raise Exception(data['error'])
    return data['body']

# Fetch a private service with no IP exposure
html = mesh_fetch('demo.gtw', token=MY_SESSION_TOKEN)
print(html[:200])

# Works with any service that has a port route registered
response = mesh_fetch('payments.gtw:8080', token=MY_SESSION_TOKEN)
data = json.loads(response)

Host Your Own Private Service

Any TCP service reachable from your machine can be exposed as a .gtw address. Nobody on the internet can reach it directly — only consumers who request it through the network.

1

Run Your Sertone Gateway

Deploy the free Sertone Docker container. This is your gateway into the network.

docker run -d --name sertone \
  -p 80:80 -p 3002:3002 \
  -v sertone-data:/app/data \
  sertonenet/gateway:latest
2

Register a .gtw Domain

In your gateway's web console, go to Mesh → Domains and register your name. Your service becomes reachable as yourname.gtw — visible only inside the Sertone network.

3

Add Port Routes

Map ports to your backend services. For example, map port 80 to localhost:3000 where your web app runs. The backend does not need a public IP — it only needs to be reachable from your gateway machine.

4

Optional: Set Pricing

If you want to charge for access, set a price per request in USDC. Free services are also supported — useful for internal tools and team access.
